Setting up Development Environment

In this book, we will be covering different kinds of projects, from basic Angular App to a Server rendered, Native App for Android and iOS. We will also be looking at how to create Component library for our enterprise applications and testing our components.

Angular Apps are written using Web Technologies and to run Angular Apps we first need Node.js to be installed on our machines.

Angular is build using TypeScript, which is a superscript of ECMAScript. It adds support for types in JavaScript, which makes the development of Angular Apps easier on compile time. We will look into some basic concepts of TypeScript in this chapter, which will clear some syntax that we will be using in this book.

Another tool we will look in here, is Angular CLI, the command line interface program that Angular Team has created, which helps in the development and maintenance of Angular Apps.

We could write Angular Apps using a simple text editor, but to get full productivity of building Angular Application we would be using VSCode, which isn’t an Integrated Development Environment (IDE) but is a code editor with rich extension support. In this chapter, we will use VSCode and install the required extensions to make Angular Development faster.

The following topics will be covered in this chapter:

  1. Understanding the basics of TypeScript
  2. Installing Angular CLI and Angular Console
  3. Installing VSCode and installing various extensions
  4. Installing Augury and seeing various sections that help us in debugging our application

Building a Flashcard game using Angular

In this chapter, we will develop a basic Angular App for a Flashcard game. We will also introduce some of the build-in Attribute and Structural directives in Angular Framework, and create a basic template driven form to add/edit the flashcards. We will get introduced to Observables and learn how to use the async pipe to get observable data on the page. We will also create a basic FlashService to do all the operations related to adding, editing, and deleting flash.

The following topics will be covered in this chapter:

  1. Different decorators in Angular like @Input, @Output, @ViewChild.
  2. Different build-in Attribute and Structural Directives in Angular
  3. How to create a simple Template Driven forms
  4. Services in Angular
  5. Importance of Observables in Angular
3

Building a Personal SPA blog using Angular Router and Wordpress

In this chapter, we will create a Personal Blog using Angular. We will create it as a Single Page Application (SPA) using Angular Router. Single Page Applications has a benefit of loading most of the application on our initial load, which gives a benefit when you browse the website a bit more. When going from one page to another, SPA doesn't have to download the whole page but only request the required API for the page, which is way faster.

The following topics will be covered in this chapter:

  1. We will add Angular Router and configure multiple routes
  2. Lazy Load our routes to make initial render faster
  3. Http calls to connect to an API and display data using it.
  4. Preloading Strategy to make the page transition faster
  5. Animations to make add meaning to our application.

Building an Inventory application using Reactive Forms

In this Chapter, we will be creating an application for managing inventory. We will be displaying products and doing CRUD (Create, Read, Update and Delete) operations. We will be introduced to Reactive Forms in this chapter, which is the best way of creating forms in Angular because you will have more control over the form as well as you can create complex forms using it.

The following topics will be covered in this chapter:

  1. Use of third party Angular Components using Clarity Components
  2. Reactive Forms
  3. Using in build validation provided by Angular
  4. Creating custom validation for your form

Building a PWA Survey Application using Angular Service Worker

In this chapter, we will be creating an e-commerce application for an electronic store, and then convert it into a Progressive Web App, add support for offline, and also able to add the web app as an application on the home screen on both Android and iOS. We will also set up a nest.js application to get data from MongoDB in this chapter.

The following topics will be covered in this chapter:

  1. We will create a nest.js application for backend service
  2. We will use Angular Material to create a simple e-commerce application
  3. We will use lighthouse to measure and analyze our application for PWA.
  4. We will convert our application to score perfect 100 on the lighthouse for PWA
  5. We will make our application run offline
  6. We will deploy our application on firebase hosting

Building an Auditing application using Angular and Ionic

In this chapter, we will be using Firebase Real-time Database to create a Native Application that runs on iOS and Android using the Ionic Framework. We will also be using ngx-formly for creating a dynamic form without having to write an HTML for the whole form.

The following topics will be covered in this chapter:

  1. Installing and creating an Ionic Application using Ionic CLI
  2. Configuring Firebase Real-time Database
  3. Getting collection from Firebase and updating it
  4. Creating a dynamic form using ngx-formly

Server Side Rendering our Personal SPA Blog using Angular Universal

In this chapter, we will continue to work on our personal blog from Chapter 3, Building Personal SPA Blog using Angular Router and Wordpress Rest API, where we created a Single Page Application using Angular Router. We will render the application on the Server so that the performance and the Search Engine Optimization can be optimized. We will use Transfer state to make sure that the APIs are cached when rendered in the Server so that the APIs are not called redundantly on the client. We will use different SEO optimizations to make it be discoverable on Search Engines and shared properly on Social media platforms like Twitter, Facebook, and LinkedIn

The following topics will be covered in this chapter:

  1. Deploy our application on Zeit Now
  2. Measure the performance of our application using the lighthouse
  3. Add Angular Universal to our application using one simple CLI command
  4. Transfer application state from Server to Client
  5. Optimize our application for Search Engines and Social Media scraping using ngx-seo.

Building an enterprise portal using Nx, NgRx and Redux

In this chapter, we will be creating an enterprise application for admins to login and check the sales of our existing e-commerce application we developed. We will be using OAuth using Auth0 to authenticate users to our application and check the sales of each product using D3 Visualizations. We will be using Nx workspace to create our monorepo application, which will include not only our e-commerce application and admin application, but also libraries that can be shared by them.

The following topics will be covered in this chapter:

  1. Upgrading an existing Angular CLI application to Nx workspace
  2. Using high-quality components from Momentum Design
  3. Authentication by Token-based Authentication using Auth0
  4. Guarding routes using Angular Guards
  5. Using JWTModule which includes HTTP interceptor
  6. Using ngx-charts to use D3 Visualization

Building Multi-language NativeScript Application with Angular

In this chapter, we are going to develop a Nativescript application along with Angular to display news postings. We will allow the user to select from a range of different languages and display news in the selected language. We will be mocking the API to fetch news post for different languages. We will use two languages – English and Spanish. We will be using ngx-translate to add internationalization to our application and end the chapter by using a nativescript plugin to use a native snackbar.

The following topics will be covered in this chapter:

  1. Creating Nativescript application and run in on Nativescript playground.
  2. Learn about different Nativescript Container Layouts
  3. Use ngx-translate to add internationalization to our application
  4. Use a nativescript plugin.

Building a Component library using Angular CDK and Angular Elements

In this chapter, we will build a component library that can not only be used in our applications but can also be published on npm so that it can be used by any other applications that want to use our component library. The reusable component libraries are something that makes teams build awesome applications super fast. We will be building a dropdown reusable component library that can be flexible enough for multiple reusabilities in the application.

The following topics will be covered in this chapter:

  1. Build a reusable dropdown component
  2. Add accessibility features to the component library
  3. Use Angular CDK to manage Focus for keyboard users
  4. Publish the component library to npm
  5. Create an Angular Element to use in any application
11

Testing Angular Application using Jasmine, Jest and Protractor

In our last chapter, we will cover one of the important topics in Software development. We will be covering about testing our Angular Application. We will start with an existing application that was built using Angular CLI, and will make sure that all the existing tests pass, and then add new Isolated test case, some Deep integration testing to our application. We will then move on to running code coverage report and setting thresholds to our coverage report. We will then move on to updating our tests to use Jest as the test runner to make the testing experience delightful. We will end this chapter by adding some End to End tests to test the application.

The following topics will be covered in this chapter:

  1. Learn about why testing is important in Software Development
  2. Understand and test using Isolated Unit testing, Deep Integration testing, and Shallow Integration testing
  3. Upgrade our Jasmine and Karma default setup to use Jest Test Runner
  4. Use protractor to run End to End testing of our application
